# AI Agent Operating Rules

このファイルは、このディレクトリで作業するAIエージェントの振る舞いを定義するものです。

## 役割

- WorkHub のプロトタイプ実装を、ユーザーと認識を合わせながら進める。
- WorkHub に関するドキュメントを整理し、実装や議論の前提が追いやすい状態に保つ。

## 最優先ルール

- 親エージェントは、ユーザーの意図を確認し、作業方針を整理し、必要な実装を統合する。
- サブエージェントは、親エージェントが明示した独立タスクだけを担当する。
- どのエージェントも、ユーザーの意図を推測だけで確定しない。
- `docs/product-concept/README.md` は原典として扱う。この方針と異なる実装や変更指示があれば、ユーザーに再確認する。
- `docs/product-concept/README.md` からでは分からない、本番の実装で必要そうな情報や判断、ユーザーと認識を合わせた内容などを `prototype-communication.md` にメモとして記録してください。本番の実装の際の参考資料となります。

## 参照順序

1. 直近のユーザー指示
2. `docs/product-concept/README.md`
3. `prototype-communication.md` の確認事項・合意事項
4. ユーザーが共有した参考ページ・画像・説明
5. 既存の実装

## 禁止事項

- `docs/product-concept/README.md` を勝手に編集すること。
- アプリ仕様を `AGENTS.md` に書くこと。
- ユーザー確認が必要な仕様を、勝手に実装で確定すること。
- 指示されていない外部ページ遷移、外部検索、外部サービスへのリンク遷移を実装すること。
- 仕様理解が不十分なまま、UI部品を増やして進捗に見せること。
