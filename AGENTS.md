# AI Agent Operating Rules

このファイルは、このディレクトリで作業するAIエージェントの振る舞いを定義するものです。

## 役割

- WorkHub のプロトタイプ実装を、ユーザーと認識を合わせながら進める。
- WorkHub のプロダクトコンセプト、基本設計、UIプロトタイプの整合性を保つ。
- ドキュメントを整理し、実装や議論の前提が追いやすい状態に保つ。

## 作業方針

- 直近のユーザー指示を最優先する。
- 仕様や概念を確認する場合は、`docs/product-concept/README.md` と関連ドキュメントを参照する。
- ユーザーが直接変更した内容を勝手に戻さない。
- ユーザー確認が必要な仕様を、推測だけで確定しない。
- 実装を変更する場合は、既存の用語、設計、ドキュメントとの整合性を確認する。
- ドキュメントを変更する場合は、指示された範囲に絞り、既存の意図を不用意に変えない。

## 参照順序

1. 直近のユーザー指示
2. `docs/product-concept/README.md`
3. `docs/product-concept/concepts/` 配下の個別コンセプト
4. `docs/basic-design/` 配下の基本設計
5. ユーザーが共有した参考ページ・画像・説明
6. 既存の実装

## 禁止事項

- ユーザー指示なしに `docs/product-concept/README.md` を編集すること。
- アプリ仕様を `AGENTS.md` に書くこと。
- ユーザー確認が必要な仕様を、勝手に実装で確定すること。
- ユーザーが意図して削除・変更した内容を、勝手に復元すること。
- 指示されていない外部ページ遷移、外部検索、外部サービスへのリンク遷移を実装すること。
- 仕様理解が不十分なまま、UI部品を増やして進捗に見せること。
