# 管理区分

管理区分とは、WorkHub で扱う仕事を性質ごとに分けるための分類である。Project / Ad-hoc / Explore はチケット管理の対象として扱い、Reserved はカレンダー上で時間を占有する予定として、実稼働時間の算出に使う。

## Project

| 項目 | 内容 |
| --- | --- |
| 定義 | ゴールがあり、まとまりがあり、複数の成果物やタスクがつながっている仕事。スプリントで計画・実行・振り返りを回す対象であり、GitHub Milestone / Project と連携する |
| 例 | 社内システムのアジャイル開発, 業務自動化推進PJ |

## Ad-hoc

| 項目 | 内容 |
| --- | --- |
| 定義 | Project に属さない単品の仕事。依頼や発生事象をきっかけに発生し、カテゴリー分けして管理する。時間と紐づく定期業務は Calendar（Reserved）で管理する |
| 例 | 単発の分析依頼, 問い合わせ対応, 軽微な調査, 細かな改善 |

## Explore

| 項目 | 内容 |
| --- | --- |
| 定義 | 自発的に取り組む学習・探究・実験・挑戦。工数管理がしづらく定量的に管理しにくい探究活動を、あえて管理区分として独立させ、見積もりと実績を追跡する |
| 例 | 新技術の学習, SaaS/API 検証, AI活用実験, 自動化の試作 |

## Reserved

| 項目 | 内容 |
| --- | --- |
| 定義 | 管理区分ではない。カレンダー上で時間を占有する予定。タスク管理の対象にはせず、実稼働時間の算出に使う |
| 例 | 全社会議, 朝会, 休暇, 移動, 定期業務（時間が固定のもの） |
