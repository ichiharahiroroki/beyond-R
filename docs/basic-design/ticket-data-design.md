# 🎫 チケット基本設計

このドキュメントは、WorkHub のチケットが持つ情報を整理する基本設計書である。チケット管理制の概念は、[チケット管理制](../product-concept/concepts/ticket-management-system.md) を参照する。

---

## 🧩 チケットの共通情報

すべてのチケットは、チケット同士の階層関係、画面表示、検索、進捗集計、外部ツール連携に必要な情報を持つ。

| 区分 | 主な情報 |
| --- | --- |
| 基本情報 | チケット名、[管理区分](../product-concept/concepts/work-management-categories.md)、チケット種別、説明、優先度、ラベル |
| 担当情報 | 担当者、アサイン |
| 日付情報 | 開始予定日、完了予定日、期限 |
| 工数情報 | 見積工数、投下工数 |
| 活動履歴 | ディスカッション、変更履歴、投下工数ログ |
| システム管理情報 | 内部ID、表示ID、親チケットID、作成日時、更新日時 |

## 🗂️ チケット種別特有の情報

| チケット種別 | 特有の情報 | 補足 |
| --- | --- | --- |
| Project | 予算、見積工数 | 配下にマイルストーンを持つ |
| マイルストーン | 達成予定日、リリース日、進捗、Project 管理区分の予算・見積工数 | 配下チケットから進捗と工数を集約する |
| カテゴリ | 進捗 | Project 以外の管理区分で、仕事の種類を大きく整理する |
| グループ | 進捗、Project 管理区分の予算・見積工数 | 必要に応じて成果物やタスクをまとめる任意階層 |
| 成果物チケット | 依頼人、進捗、Project 管理区分の予算・見積工数 | 配下タスクから進捗と工数を集約する |
| タスクチケット | ステータス、Project 管理区分の経費、投下工数ログ | 実際に作業・レビューする最小単位 |

## 🧭 チケット情報の扱い方

| 区分 | 情報 | 対象 | 扱い | 説明 |
| --- | --- | --- | --- | --- |
| 基本情報 | チケット名 | 全チケット | 人が入力 | チケットの内容が分かる名前を入力する |
| 基本情報 | 管理区分 | 全チケット | 人が選択 | Project / Ad-hoc / Explore から選ぶ |
| 基本情報 | チケット種別 | 全チケット | 人が選択 | 作成できる種別はチケット階層ルールで制御する |
| 基本情報 | 説明 | 全チケット | AIが生成・集約 / 人が編集 | Markdown で自由に記述する。テンプレート選択、関連チケット、ディスカッション、変更履歴などをもとに、AI が下書きや要約を補佐できる |
| 基本情報 | 優先度 | 全チケット | 人が選択 | `低`、`中`、`高`、`クリティカル` から選ぶ |
| 基本情報 | ラベル | 全チケット | 人が選択 | ユーザーが任意に作成したラベルを複数選択できる |
| 担当情報 | 担当者 | 全チケット | 人が選択 | チケットの作業を担当する人を選ぶ |
| 担当情報 | アサイン | 全チケット | 人が選択 | 現在ボールを持っている人を選ぶ |
| 日付情報 | 開始予定日 | 全チケット | 人が入力 | 作業を開始する予定日を入力する |
| 日付情報 | 完了予定日 | 全チケット | 人が入力 | 作業を完了する予定日を入力する |
| 日付情報 | 期限 | 全チケット | 人が入力 | チケットの期限を入力する |
| 工数情報 | 見積工数 | タスクチケット | AIが算出 / 人が調整 | AI がタスク単位で初期見積を出し、人が必要に応じて調整する |
| 工数情報 | 見積工数 | 成果物以上のチケット | システム計算 | 配下タスクチケットの見積工数を合計する |
| 工数情報 | 投下工数 | タスクチケット | 人が入力 | 実際に使った作業時間を投下工数ログとして入力する |
| 工数情報 | 投下工数 | 成果物以上のチケット | システム計算 | 配下タスクチケットの投下工数を合計する |
| 活動履歴 | ディスカッション | 全チケット | 人が入力 | チケットごとの議論や確認事項を残す |
| 活動履歴 | 変更履歴 | 全チケット | システム記録 | 誰が、いつ、どの項目を、どのように変更したかを記録する。ステータス変更から実際に開始・完了したタイミングも確認できる |
| 活動履歴 | 投下工数ログ | タスクチケット | 人が入力 | 投下日と投下時間を記録する |
| 種別特有情報 | ステータス | タスクチケット | 人が選択 | `todo`、`doing`、`review`、`done`、`discard` の5つを使う |
| 種別特有情報 | 進捗 | タスク以外のチケット | システム計算 | 配下タスクチケットのうち、`done` になっている割合で示す |
| 種別特有情報 | 依頼人 | 成果物チケット | 人が選択 | 成果物の依頼元を選ぶ |
| 種別特有情報 | 達成予定日 | マイルストーン | 人が入力 | マイルストーンを達成する予定日を入力する |
| 種別特有情報 | リリース日 | マイルストーン | 人が入力 | リリース予定日を入力する |
| Project 管理区分 | 予算 | Project / マイルストーン / グループ / 成果物チケット | 人が入力 | Project の予算を上限とし、下位チケットの予算は上位チケットの予算を超えて入力できない |
| Project 管理区分 | 経費 | タスクチケット | 人が入力 | タスクチケットに経費を入力する |
| Project 管理区分 | 経費合計 | 成果物以上のチケット | システム計算 | 配下タスクチケットの経費を合計し、予算と比較して表示する |
| システム管理情報 | 内部ID | 全チケット | システム管理 | システム内部の参照に使う |
| システム管理情報 | 表示ID | 全チケット | システム管理 | 画面表示、検索、会話、URL共有で使う |
| システム管理情報 | 親チケットID | 子チケット | システム管理 | チケット階層を管理するために使う |
| システム管理情報 | 作成日時 | 全チケット | システム記録 | チケット作成時に自動記録する |
| システム管理情報 | 更新日時 | 全チケット | システム記録 | チケット更新時に自動記録する |

## 🧱 チケット階層ルール

チケットは、[管理区分](../product-concept/concepts/work-management-categories.md) とチケット種別に応じて、作成できる親子関係を制御する。

### Project

```text
Project
  └ マイルストーン
      └ グループ
          └ 成果物チケット
              └ タスクチケット
```

Project では、Project の配下にマイルストーンを作成する。マイルストーンと成果物チケットの間には、必要に応じてグループを作成できる。グループは任意であり、複数階層にできる。

### Project 以外

```text
管理区分
  └ カテゴリ
      └ グループ
          └ 成果物チケット
              └ タスクチケット
```

Ad-hoc や Explore では、管理区分の配下にカテゴリを作成する。カテゴリと成果物チケットの間には、必要に応じてグループを作成できる。グループは任意であり、複数階層にできる。

## 📊 集約の考え方

上位チケットの進捗と工数は、原則として配下のチケットから集約する。

| 集約対象 | 集約元 | 考え方 |
| --- | --- | --- |
| 進捗 | 配下チケットの完了状況、見積工数、投下工数 | 上位チケット単体ではなく、配下の進み具合から判断する |
| 見積工数 | 配下タスクの見積工数 | 成果物以上は、配下タスクの見積工数を合計して扱う |
| 投下工数 | 配下タスクの投下工数 | 実際の作業時間はタスクチケットに記録し、上位へ集約する |
| 経費 | 配下タスクの経費 | Project 管理区分では、成果物以上のチケットで配下タスクの経費合計と予算を比較して表示する |

---

## 🔑 IDの扱い

チケットには、システム内部で参照するための内部IDと、人間が画面上で参照するための表示IDを持たせる。また、チケット階層を管理するために親チケットIDを持たせる。

内部IDは、親子関係、コメント、変更履歴、投下工数、外部リンクなどの紐づけに使う。表示IDは、画面表示、検索、会話、URL共有など、人間がチケットを指し示すために使う。

---

## ⚙️ システム管理情報

内部ID、表示ID、親チケットID、作成日時、更新日時は、ユーザーがチケット内容として入力する情報ではなく、システムが管理するメタ情報として扱う。
