# 🧮 見積もり基本設計

このドキュメントは、WorkHub におけるタスクチケットの見積工数算出を、実装に近い基本設計として整理するものである。概念上の基準は、[見積もり基準](../product-concept/concepts/estimation-criteria.md) を参照する。

---

## 🎯 目的

タスクチケットの見積工数を、AI が初期算出し、人間が必要に応じて調整できるようにする。

見積工数は、担当者ごとの実力差を含めない。タスクそのものの大きさを、基準実力を持つ人が実行した場合の時間として扱う。

---

## 📌 対象範囲

このドキュメントで扱う見積工数は、タスクチケットの作業見積と、その集約値である。Project 管理区分で扱う予算・経費との比較は、[チケット基本設計](./ticket-data-design.md) に従う。

| 対象 | 扱い |
| --- | --- |
| タスクチケットの見積工数 | AI が初期見積もりを行い、人間が調整・確定する |
| 成果物以上の見積工数 | 配下タスクチケットの見積工数を集約する |
| 予想戦闘力 | [リソース管理の考え方](../product-concept/concepts/resource-management.md) に基づいて扱う |
| 投下工数 | タスクチケットに記録し、成果物以上へ集約する |

---

## 🔁 見積もりの流れ

```text
タスクチケット作成
  ↓
AI がチケット情報と関連文脈を読む
  ↓
AI が見積工数、見積根拠、不足情報、分解提案を出す
  ↓
人間が確認し、必要に応じて見積工数を調整する
  ↓
確定した見積工数をタスクチケットに保存する
  ↓
成果物以上のチケットへ見積工数を集約する
```

---

## 📥 AI見積もりの入力情報

| 区分 | 情報 | 内容 |
| --- | --- | --- |
| チケット基本情報 | 表示ID、チケット名、管理区分、チケット種別、優先度、ラベル | 対象チケットを識別し、作業の性質を判断する |
| 説明 | Markdown本文 | 目的、背景、作業内容、完了条件、レビュー観点、関連リンクなどを読む |
| 階層情報 | 親チケット、上位チケット、配下チケット | Project、マイルストーン、グループ、成果物との関係を読む |
| 活動履歴 | ディスカッション、変更履歴 | 過去の判断、確認事項、仕様変更を読む |
| 関連文脈 | 関連チケット、外部ツールへのリンク、AIセッション | チケット単体では分からない背景を読む |
| 対象範囲 | 対象画面、対象機能、対象ファイル | どこを変更・確認する作業かを判断する |

外部ツールの内容は、原則としてチケットの説明や関連リンクから参照する。外部ツールそのもののデータ取得方法は、この基本設計では扱わない。

---

## 📤 AI見積もりの出力情報

| 出力 | 保存先 | 内容 |
| --- | --- | --- |
| 見積工数 | タスクチケットの見積工数 | 基準実力を持つ人が実行した場合に必要な時間 |
| 見積根拠 | 説明または変更履歴 | どの情報をもとに見積もったか |
| 不足情報 | 説明またはディスカッション | 見積もりの確度を下げている未確認事項 |
| 分解提案 | 説明またはディスカッション | タスクが大きすぎる場合の分割案 |
| AIセッション情報 | 活動履歴 | どのAI見積もり結果を使ったかを追える情報 |

---

## 🛠️ 人間による調整

AI の初期見積もりは確定値ではない。人間は、既存コードの状態、レビュー方針、外部依存、実装上の制約、チケット分解の粒度を踏まえて、見積工数を調整できる。

見積工数を調整した場合は、変更履歴に以下を残す。

| 情報 | 内容 |
| --- | --- |
| 変更者 | 見積工数を変更した人 |
| 変更前の見積工数 | 変更前の値 |
| 変更後の見積工数 | 変更後の値 |
| 変更理由 | なぜ調整したか |
| 変更日時 | いつ変更したか |

---

## 📊 集約ルール

タスクチケット以外の見積工数は、原則として配下タスクチケットの見積工数を合計して算出する。

```text
上位チケットの見積工数 = 配下タスクチケットの見積工数の合計
```

グループや成果物の階層が複数あっても、最終的には配下にあるタスクチケットの見積工数を集約する。

---

## ✅ バリデーション

| ルール | 内容 |
| --- | --- |
| 見積対象 | AI が見積工数を算出する対象はタスクチケットとする |
| 上位チケット | 成果物以上の見積工数は配下タスクチケットから集約する |
| 未確定情報 | 情報不足の場合は、見積工数を無理に確定せず、不足情報を出す |
| 分解基準 | タスクが大きすぎる場合は、分解提案を出す |
| 変更履歴 | 見積工数を変更した場合は、変更履歴に残す |
